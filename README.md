# Mac Cleaner MVP

一个本地运行的 macOS 垃圾清理桌面原型，目标是把“看懂空间占用、保守扫描、默认安全清理”这条主流程先跑通。

## 定位

这不是系统优化工具，也不做激进清理。

当前版本只关注两件事：

- 把常见低风险垃圾数据找出来
- 用尽量保守的方式执行清理

## 风险分级规则

- 低风险：系统缓存、应用缓存、日志文件、废纸篓
- 中风险：下载目录旧安装包、下载目录大文件、常见目录大文件
- 高风险：当前版本不做真实扫描和删除，只显示保守占位说明

## 扫描范围

当前规则扫描只覆盖这些路径：

- `/Library/Caches`
- `~/Library/Caches`
- `/Library/Logs`
- `~/Library/Logs`
- `~/.Trash`
- `~/Downloads`
- `~/Desktop`
- `~/Documents`
- `~/Movies`
- `~/Music`
- `~/Pictures`

当前大文件扫描会跳过一些明显噪声目录，例如：

- `.git`
- `node_modules`
- `build`
- `dist`
- `coverage`
- `Library`

这样做的目的是先保证安全和可解释性，而不是尽可能扫得深。

## 本地运行

环境要求：

- macOS
- Node.js 20 或更新版本
- npm

安装依赖：

```bash
npm install
```

启动开发环境：

```bash
npm run dev
```


- `out/renderer`

## 权限与安全说明

- 当前版本是本地应用，不上传扫描结果，不接远程服务
- 扫描策略默认保守，不做全盘深度分析
- 低风险项才会默认勾选
- 中风险项需要用户手动确认
- 高风险项当前不会进入清理计划
- 清理动作默认是“移到废纸篓”，不是永久删除
- 对于已经在废纸篓中的项目，清理前会要求额外确认
- 如果某个目录没有访问权限，扫描会跳过并记录原因，不会中断整个流程

如果扫描结果明显偏少，通常优先检查：

- 应用是否拥有足够的磁盘访问权限
- 目标目录是否确实存在
- 当前账户是否有权限读取对应目录

## 当前测试覆盖

当前测试集中覆盖这些内容：

- 扫描分类是否保持保守顺序和默认勾选策略
- 风险等级是否按预期分配
- 扫描根目录是否按规则映射
- 高风险占位类目是否保持只读语义
- 旧安装包识别规则
- 大文件阈值规则
- 大文件遍历跳过规则
- 空分类结果构造
- 清理计划只包含可操作类目
- 废纸篓清理是否要求显式确认
- 空清理计划是否安全结束
- 高风险和不存在类目是否被排除在清理外

## 手动验收清单

可以按下面顺序做一次 MVP 验收：

1. 运行 `npm run dev`
2. 确认首页能展示磁盘容量、健康状态和开始扫描按钮
3. 点击开始扫描，确认能进入扫描中页面并看到进度更新
4. 扫描完成后确认结果页能展示分类、大小、风险等级和默认勾选状态
5. 只选择低风险项执行清理，确认能进入清理进度页并完成
6. 如果选择了废纸篓类目，确认会出现额外确认
7. 清理完成后确认释放空间和失败项能正常展示
8. 重启应用，确认设置和最近一次扫描摘要仍然存在

## 当前限制

- 这是 MVP，不做重复文件、浏览器缓存、启动项优化等扩展能力
- 高风险项目前只是占位，不做真实扫描和删除
- 永久删除选项在设置中仅作保留，不正式开放
- 下载目录和大文件识别还是基础版规则，不做复杂智能分析
- 目前不支持取消扫描、暂停清理或任务恢复
- 当前测试以规则与清理计划为主，没有做端到端 UI 自动化
